# 项目范围变更记录（2026-07-22）

## 决策

项目正式从“依赖已有激光全局地图、访问预设观察点的文字寻物导航”调整为：

> 在目标已进入当前视野并连续确认的条件下，利用 RGB-D、短时里程计、实时激光和滚动局部代价地图，完成有限距离的局部路径规划、静态避障接近和安全停止。

## 删除的依赖与承诺

- 不建立、加载或依赖预建全局地图。
- 不要求 `map` 坐标系、AMCL 或全局定位。
- 不访问 4—8 个地图观察点。
- 不在看不到目标时自主向前探索。
- 不承诺跨房间搜索、长走廊导航、绕长墙或复杂迷宫能力。

## 保留的必要能力

- `odom → base_link → camera_link` 短时 TF 和里程计健康状态。
- 当前视野目标连续三帧确认和持续重定位。
- 深度/点云与实时激光生成滚动局部代价地图。
- 地面/离群点过滤、射线清空、0.5—1 s 短时记忆和障碍膨胀。
- 局部停靠点、短路径、分段执行和滚动重规划。
- 急停、触边、抱闸、人工接管、声光报警、激光停障及数据/里程计异常停止。

## 失败策略

- 当前视野未确认目标：`TARGET_NOT_VISIBLE/FAILED`，不产生导航调用。
- 接近期间目标丢失：立即停止，重新确认后才能继续。
- 局部窗口无安全路径：`LOCAL_PATH_BLOCKED`，不猜测窗口外路线。
- 相机、深度、TF、里程计、控制接口或安全状态未知/过期：`STOP/FAILED`。
