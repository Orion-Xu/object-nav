# AGENTS.md

## 项目概述

本项目是一个为期一个月、由第一次接触机器人视觉与导航的实习生独立完成的小型项目。目标是在屈膝升降式移动双臂/四臂机器人平台上，使用图漾科技 GM461-E1 实现“目标可见条件下的短距离局部导航与静态避障”Demo。

默认流程为：输入“寻找椅子”“找一下背包”等文字指令，解析为可配置的规范物体名称；只有目标已在当前视野中连续确认，才使用开放词汇检测和 GM461-E1 深度计算目标相对位置，并通过持续更新的滚动局部代价地图短距离接近目标。系统不依赖预建全局地图，不执行跨房间搜索。

这里的“泛化”是受控的常见物体泛化，不等于任意自然语言、任意物体、完整 VLN 或未知环境探索。首版采用可编辑词表与物体策略；增加预训练模型能够识别的常见类别时，优先只修改配置。GM461-E1 负责目标检测、三维定位和局部障碍感知，实时激光可作为滚动局部图和平台安全链路的补充，但不用于建立项目所依赖的全局地图。

## 事实来源与优先级

平台能力、交付边界和安全条件以《26014-MCM2609-屈膝式类人双臂_四臂机器人（V2-263）.docx》及现场确认记录为依据。发生冲突时遵循：补充协议/会议纪要/澄清邮件 > 技术协议 > 方案 > 本文件；同级文件以较新版本为准。

GM461-E1 已进入项目范围，但 RGB、Depth、CameraInfo、RGB-D 对齐、固件、SDK/ROS 驱动、深度单位、时间戳、安装外参和网络配置必须在第一周用 Percipio Viewer、官方示例和现场数据确认。未实测前，不写死分辨率、帧率、量程、坐标轴方向或深度比例。

参数表主体规格为双臂版本。实际臂数以交付配置和现场验收为准，不在程序中写死。底盘描述同时出现双轮差速和横移能力，首版只规划前进速度与角速度；只有厂商明确接口并完成安全测试后才启用横移。

## 当前平台约束

- 机器人尺寸约 1050 mm × 700 mm × 1560–1815 mm，升降行程约 250 mm，自重约 250 kg。
- 最高速度约 0.8 m/s，实习 Demo 必须从远低于最高速度的受控低速开始。
- 地面平整度、坡度、越障、越沟和通行宽度不得超出技术参数表边界；通行宽度至少约 1000 mm，并额外考虑机械臂姿态和人员余量。
- 急停、前后安全触边、抱闸、人工接管、声光报警、激光停障和定位异常停机始终优先于视觉与导航建议。
- 项目不依赖激光 SLAM 全局地图；仍需短时里程计、`odom → base_link → camera_link` TF、实时激光停障和平台安全能力。
- 标准避障为停障；减速距离约 0.5 m 且应配置化。高空避障默认不存在。
- 工控机已知为 32G+256G、预留千兆网口，GPU 未确认，因此主线按 CPU 可运行设计，不承诺固定实时帧率。

## 项目目标与最低验收

最低交付必须在录制 RGB-D 或可控模拟输入上完成完整闭环：

1. 本地界面接收“寻找/找/去找/帮我找 + 常见物体”形式的中文指令。
2. `command_parser` 清洗文字，通过 YAML 中的中文别名映射到规范英文提示词和物体策略；不让自由文本直接进入运动控制。
3. `target_detector` 使用 YOLO-World 的离线词表检测常见物体，输出类别、置信度、边界框和模型/词表版本。
4. 首版词表示例覆盖约 15–25 种室内常见物体，如椅子、桌子、沙发、盆栽、背包、手提箱、箱子、垃圾桶、瓶子、杯子、书、笔记本电脑、键盘、鼠标、手机、时钟、花瓶和玩具熊；最终以现场模型验证结果为准。
5. 新增已验证的常见类别时，只修改词表、中文别名和安全策略，不修改解析代码；配置热重载或重启后生效。
6. 检测框中央区域有效深度的中位数用于距离估计，经 `camera_link → base_link → odom` 生成短时局部目标坐标；数据不同步、深度不足或 TF 缺失时不生成导航目标。
7. 当前视野没有目标或连续确认失败时直接结束任务，不通过导航寻找目标；同一候选连续三帧满足类别、置信度、深度和位置稳定性才确认。
8. 根据目标相对位置计算约 1 m 的可配置停靠位姿；连续深度/点云和实时激光形成带 0.5–1 s 短时记忆的滚动局部代价地图，并按短距离滚动窗口反复重规划。
9. 所有状态、检测、深度、局部目标、局部路径、障碍更新、导航结果、安全决策和停止原因均可记录、回放和解释。

只有 RGB-D、定位、导航/RCS 写接口及全部安全条件和现场审批均通过后，才进行低速实机闭环；否则以 Mock 适配器、路径可视化或离线回放验收。

## 模型、词表与物体策略

- 主线检测器为 `YOLOv8s-worldv2`（YOLO-World），通过 `set_classes`/离线词表加载本次任务的英文物体提示词。先在 PyTorch 上验证，再根据现场环境测试可用的导出与 CPU 推理方式。
- CPU 性能不足、模型导出或运行环境不兼容时，降级为 `YOLOv8n` + ONNX Runtime，只启用其预训练类别中经过验证的常见物体子集。降级必须在界面和日志中显示，不得假装仍支持全部开放词表。
- CLIP ViT-B/32 仅作为默认关闭的候选复核开关；可在多个候选之间低频排序，不负责产生目标框，也不进入安全循环。
- 不接云端大语言模型。解析器只处理受控动作模板、中文别名和规范目标；不能可靠映射的文字返回“未配置/请从支持列表选择”，不启动运动。
- 词表配置至少包含：`canonical_label`、`aliases_zh`、`prompt_en`、`navigation_mode`、`min_confidence`、`min_valid_depth_ratio` 和可选 `standoff_m`。
- `navigation_mode` 取值为 `approach`、`detect_only` 或 `disabled`。大而稳定的静态物体可设为 `approach`；瓶子等小型、反光或透明物体只有深度质量达标时才允许靠近；人员和危险/不确定类别只能 `detect_only` 或 `disabled`。
- 新词进入 `approach` 前，必须完成离线图片检测、连续帧稳定性、深度有效性、停靠距离和路径安全测试。识别得到一个名称不自动获得运动权限。
- 任何视觉模型结果都不是安全信号。低置信度、无效深度、数据超时或模型故障只能导致继续搜索、只显示检测、减速、停止或失败。

## 推荐模块

- `platform`：设备型号、实际臂数、传感器、软件版本、能力开关和现场确认结果。
- `command_ui`：本地 Tkinter 输入、支持物体列表、当前状态、错误和停止按钮；不直接控制底盘。
- `vocabulary_manager`：加载、校验和版本化常见物体词表及物体策略。
- `command_parser`：中文动作模板与别名匹配，输出规范目标和导航许可；拒绝未配置或禁用目标。
- `camera_adapter`：读取 GM461-E1 RGB、Depth、CameraInfo、点云和时间戳。
- `target_detector`：YOLO-World 离线词表推理；必要时切换 YOLOv8n 降级后端。
- `target_verifier`：可选 CLIP 候选复核，默认关闭。
- `target_localizer`：目标框深度统计、反投影和坐标变换，输出 `PoseStamped` 或等价结构。
- `search_manager`：只维护当前可见范围内的连续三帧确认，不产生观察点导航。
- `local_mapping`：连续深度/点云形成滚动局部栅格或代价地图。
- `navigation_adapter`：封装平台已有的短距离局部控制器或厂商 RCS 的里程计、局部目标、状态查询与停止请求。
- `safety_arbiter`：对安全 IO、数据时效、定位、导航和视觉质量做最高优先级仲裁。
- `telemetry`：记录指令、词表版本、模型、检测、深度、坐标、路径、安全决策和异常原因。

## 状态机

主状态为：

`IDLE → PARSE → LOCAL_SEARCH → TARGET_CONFIRMED → LOCAL_PLAN → APPROACH → ARRIVED`

- `PARSE` 同时检查目标是否在词表、检测后端是否支持、`navigation_mode` 是否允许运动；不满足时进入 `FAILED` 或只执行检测展示。
- 单帧检测不得从 `LOCAL_SEARCH` 直接进入 `APPROACH`，必须连续三帧确认；当前视野无目标时进入 `FAILED`，不自主探索。
- `LOCAL_PLAN` 只在滚动局部窗口内生成停靠点和下一段短路径；窗口内无解进入 `FAILED/STOP`，不规划窗口外路线。
- 相机/定位/导航超时、TF 缺失、安全状态未知、急停或接口故障进入 `STOP/FAILED`。
- 接近过程中目标丢失时先停止或保持安全状态，再重新确认，不使用过期目标坐标继续盲行。
- `ARRIVED` 仅表示在安全停靠距离内完成 Demo，不表示抓取、操作或安全认证完成。

## 关键接口与配置

ROS 1 Noetic 优先订阅：`sensor_msgs/Image`、`sensor_msgs/CameraInfo`、`sensor_msgs/PointCloud2`、`sensor_msgs/LaserScan`、`nav_msgs/Odometry` 和 `tf2_ros`。局部控制写接口必须采用平台已经验证的控制器或厂商 RCS；本项目不因取消全局地图而自行绕过底盘安全层发布速度。

厂商 RCS 适配层至少提供：`get_pose()`、`navigate_to_pose(goal)`、`get_navigation_status(task_id)`、`request_stop(reason)`。所有消息带时间戳、坐标系、单位和有效性；控制请求不得绕过底层安全系统。

目标词表、别名、物体策略、检测阈值、确认帧数、最大局部距离、停靠距离、RGB-D 最大时间差、数据超时、局部图尺寸/分辨率/保留时间、障碍膨胀和速度限制必须集中配置并版本化，禁止散落魔法数字。

## 实现与实机安全红线

- 项目主线不建立、加载或依赖全局地图。滚动局部障碍表示在导航执行期间必须持续更新；局部窗口内无安全路径时停止，不猜测窗口外路线。
- 检测可以低频运行，但局部障碍与安全状态更新不得被 YOLO-World、CLIP 或词表更新阻塞。
- 彩色与深度不同步、目标深度不足、内参缺失或局部 TF 不可用时，不生成局部导航目标。
- 视觉模型不得直接发送 `cmd_vel` 或等价电机指令，只能产生候选目标和导航建议。
- 人员不作为自主接近目标；人员场景继续由激光和平台安全链路优先处理，不实施动态人员自主绕行。
- 实机前逐项确认两个急停、安全触边、抱闸、人工接管、声光报警、激光停障和定位异常停机。任一项未确认不得运动。
- 不因演示关闭任何安全机制，不在源码、日志或提交中记录设备凭据、内部网络地址、客户信息或敏感图像。

## 测试与记录

最低测试集：

- 指令泛化：至少 8–10 种配置常见物体及不同中文表达能够映射到规范提示词。
- 配置扩展：增加一个模型可识别的新常见类别及别名后，不修改代码即可完成解析和离线检测。
- 安全策略：`detect_only`/`disabled` 目标不会生成导航任务；人员指令不得触发自主接近。
- 后端降级：YOLO-World 不可用时明确切换 YOLOv8n 支持子集，并正确拒绝超出子集的类别。
- 可见目标确认：无目标时不产生运动；单帧误检不触发靠近；连续三帧有效才确认。
- 深度与 TF：无效深度、时间不同步、有效像素不足或 TF 缺失时不输出目标位姿。
- 障碍与异常：局部路径被阻、相机超时、里程计丢失、目标丢失或接口失败均进入预期的减速、停止或 `FAILED`。
- 回放一致性：相同数据、词表版本和配置得到相同目标、搜索与安全结果。

每项测试记录设备与软件版本、模型/词表版本、数据来源、参数版本、预期行为、实际行为、状态变化、异常码、日志和录像/截图。

## 变更协作

- 修改前阅读相关接口、配置和测试，保持变更聚焦，不覆盖他人的未提交修改。
- 硬件能力、模型后端、词表、物体策略、坐标系、接口协议或安全参数变化时，同步更新能力清单、配置说明和测试记录。
- 发现可能造成非预期运动、绕过安全限制、超出机械边界或泄露敏感信息的问题时，停止扩大范围，优先修复或上报风险。
